Odyssey Test Mode

With this utility the user can test and tune the various features of Odyssey image processing functionalities.

  • Active Program: selects the image processing program to be used for the next tests.
  • Find Clusters button: synchronously finds the clusters in the present image. The number of clusters found is shown in the row below.
  • Max Clusters : size of the memory space to be allocated for clusters; if clusters exceed this buffer, the cluster finding test fails with a return value of 0.
  • Compute Frame Rate: if this check box is enabled, when Find Clusters is pressed, several cycles are executed, so the sustainable frame rate in synchronous mode is computed. This value depends on the camera clock, the image processing workload, the number of clusters in the image, and the Windows thread timeslice. For cameras faster than 60 fps, the output of this measurement is likely to be just a fraction of the camera frame rate, since frames are systematically missed (typically, the synchronous processing frame rate is 2/3 or 1/2 of the camera frame rate).
  • Dump Clusters: dumps the list of the clusters found in the last frame (synchronous mode) or the list of clusters in all frames (asynchronous mode). The file path is specified in the edit box on the right, or can be selected interactively by clicking the adjacent button. In order for this command to make sense, at least one clustering process must have been executed. The file formats are listed below:
    Synchronous mode: 0 Count 0 ID X Y Area InertiaXX InertiaYY InertiaXY
    Asynchronous mode: Image Count Time ID X Y Area InertiaXX InertiaYY InertiaXY
    The asynchronous mode format is compatible with the synchronous mode format.
  • Dump Image: dumps the image taken in synchronous mode from which the last bunch of clusters has been produced. No image is dumped if the Find Clusters button has not been pressed previously. This function is intended for comparison studies with other image processing sequences; dumping the image along with its corresponding clusters allows one to check the performance of the whole image processing chain in offline mode. The image is dumped in 8 bit Windows BMP format to the file specified in the edit box on the right. The user can select the file path interactively by clicking the adjacent button.
  • Asynchronously find clusters: runs the asynchronous image processing chain for the specified program, and displays timing information. In this case, clusters cannot be shown and their count is not displayed, but they can be saved to file. In this working mode, the camera frame rate is always sustained (provided it is not too fast for the grabbing hardware). The timing information can be interesting to check the effect of splitting the clustering procedure over several processors, and to study how the clustering times vary with the number of grains in the images.
  • Images: the number of images to be taken in asynchronous mode. 
  • Show Clusters: shows/hides the clusters found in synchronous mode on the secondary display. For each cluster, a circle is drawn with the center in the cluster center of gravity and with a radius increasing with increasing cluster area.
  • Make Flat Field Image: builds an .r8 file with a flat field compensation image by averaging several images. For best results, the images should be taken outside of the emulsion, exactly at the gray level that will be used in normal operation.
  • Images to use : the number of images to be averaged to estimate the flat field image. 
  • Gray Level: estimates the gray level and saturation on the present image.
  • Test Automatic Light Control: starts the automatic light control procedure . This should be done once before optimizing the image processing parameters, to ensure that the real operating conditions are reproduced during setup.
  • Inquire: inquires some hardware features. Presently the maximum number of asynchronous frames and the actual camera frame rate are returned. The first depends on the RAM configuration of the Matrox Odyssey and of the host PC; the second is measured by taking several images and averaging the cycle times.
  • Exit: exits and returns to the Edit Configuration mode.
