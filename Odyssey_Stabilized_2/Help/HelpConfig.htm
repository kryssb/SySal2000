Odyssey Configuration

Odyssey has several parameters to control the various stages of image processing. Here below the configuration dialog box is shown with very typical (not necessarily best) settings.

  • Program tab control: here the user selects the processing program to modify. At the beginning, with an empty configuration, the tab is empty also. The user must click on <New> to create a new program (see below)
  • Kernel Width / Kernel Height: the size of the filter kernel for each program; these numbers range from 1 to 8.
  • Kernel coefficients: the coefficients in the kernel matrix; since it is assumed to be symmetric on both axes, only the upper-left corner needs to be specified. In case of odd width(height), the central column(row) must be specified too. For high-pass filters, the sum of all the coefficients in the matrix must be 0; for gray level thresholding, the kernel must be 1x1, coefficient = -1.
  • Cutoff: filter threshold that discriminates interesting pixels from background pixels. Positive for high-pass filters, negative for gray level thresholding.
  • X offset: X coordinate in pixels of the left edge of the clustering window. For correct operation of the clustering module, this must be a multiple of 8 (binarized pixels are mapped to bits and are therefore packed in groups of 8).
  • Y offset: Y coordinate in pixels of the top edge of the clustering window.
  • Window Width: the width in pixels of the clustering window. For correct operation of the clustering module, this must be a multiple of 8 (binarized pixels are mapped to bits and are therefore packed in groups of 8).
  • Window Height : the height in pixels of the clustering window. 
  • Flat Field: full path of the file that contains the flat field image. This file is in .r8 format, i.e., the file contains only the signed gray levels of the pixels in the image without any header, and each pixel is in 8 bit format (1 byte per pixel). The user is responsible for supplying a file with the correct size and meaning.
  • Flat Field button: helps the user to select a valid file path by a standard file selection dialog.
  • Filter Equalization: full path of the file that contains the filter equalization image. This file is in .r16 format, i.e., the file contains only the signed gray levels of the pixels in the image without any header, and each pixel is in 16 bit format (2 bytes per pixel). The user is responsible for supplying a file with the correct size and meaning.
  • Filter Equalization button: helps the user to select a valid file path by a standard file selection dialog.
  • Extended Operations List: lists the programmed extended operations. The labels S.T, S.L, S.W, S.H, D.T, D.L, D.W, D.H stand respectively for (Source; top, left, width, height and Dest; top, left, width, height) and define the source and destination buffers for each operation.
  • Add: adds a new extended operation with the source and destination buffers specified in the edit boxes and with the currently selected type in the type combobox.
  • Set: changes the currently selected extended operation to match the source and destination buffers specified in the edit boxes and the currently selected type in the type combobox.
  • Copy: copies the currently selected extended operation to the source and destination buffers specified in the edit boxes and sets the currently selected type in the type combobox accordingly.
  • Delete: deletes the currently selected extended operation.
  • Min Control Level: the minimum light output (in lamp control units, whose range and meaning depend on the lamp controller).
  • Max Control Level: the maximum light output (in lamp control units, whose range and meaning depend on the lamp controller).
  • Lamp Time Constant: the time constant of the lamp in ms; since the lamp output depends on the lamp temperature, it is assumed that when the control signal is changed, the light output changes to the new value according to an exponential relaxation law.
  • Target Gray Level: the desired average gray level on the image.
  • Saturation Gray Level: the minimum gray level a pixel should have to be considered in saturation; this is not necessarily 255.
  • Target Saturation Fraction: when saturation occurs, the desired fraction of the image that should be saturated. Of course, this kind of control overrides the Target Gray Level; presently it is ignored, since this function is not implemented.
  • Refresh Cycle Time: the minimum amount of time between frames in a live display. Fast cameras output many more frames than a normal monitor can handle (usually no more than 60-85), so only some frames are picked up for display.
  • Clustering Processors: the number of processors to use for asynchronous clustering (synchronous clustering is always on 1 processor); if the machine does not have so many processors, the clustering threads are arranged to work on the available processors, but this turns out to be an unneeded overhead.
  • Test button: starts the test mode. The present Odyssey configuration is saved, the configuration in this dialog is applied for testing, and the test utility runs with this provisional configuration; upon exit, the saved Odyssey configuration is restored.
