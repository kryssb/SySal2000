Odyssey Overview

Odyssey represents the interface between SySal 2000 and the Matrox Odyssey vision processor. It is much more than a driver library, since it includes many high level functions and interfaces.

The specific interfaces implemented in this module are:

  • IFrameGrabber: basic image acquisition and graphics functions.
  • IVisionProcessor: synchronous filtering, image processing and cluster recognition functions.
  • IVisionProcessor2: multiple filtering support for synchronous acquisition.
  • IVisionProcessor3: asynchronous image processing and cluster recognition functions.
  • IAutomaticLightControl: support for self-adjustment of light (requires connection to a stage control module).

Most of these functionalities are intended for use by scanning driving modules. Nevertheless, some of them can be used directly in interactive mode, mainly for setup purposes, but also to help developers improve image handling software.

Odyssey can apply several processing steps to the images supplied by the camera. You can click on the highlighted text to be directed to a page containing detailed discussion. In order of execution, we have:

  • Flat field subtraction (optional): reduces the effect of electronic noise and nonuniform lighting.
  • Extended operations (optional):  image regions can be copied, flipped or rotated, so that defective areas can be masked and camera input is corrected.
  • 2D FIR (Finite Impulse Response) Filtering: a digital filter prepares the image for the clustering algorithm and cleans unwanted features.
  • Post-filtering equalization (optional):  compensates lighting and astygmatism effects by reshaping the output of the digital filter.
  • Cluster finding: clusters of interesting pixels are recognized and represented in mathematical form. Applies both to simple objects such as silver grains in the emulsion and to complex shapes such as calibration grid marks.

Odyssey can store up to 8 programs. Each program is a coherent set of parameters and settings. It is possible to switch dynamically from one program to another. In common practice, up to 4 programs are used in a scanning session: one is to acquire calibration marks, one is for top side, one is for bottom side, and the fourth one can be allocated for focusing purposes.

Cluster finding is performed by the PC CPU(s). The clustering algorithm can profit of the presence of several processors, and usually the clustering speed scales linearly with the number of processors involved. Multiprocessor clustering is activated only in asynchronous mode (IVisionProcessor3 functions).

As mentioned above, Odyssey can automatically adjust the light. In order to do this, it must drive a lamp, which is achieved if an external stage control module is connected to the LampController slot (the only connection slot featured by Odyssey). During Automatic Light Control, none of the image processing operations is used.

The SySal icon for this module is a photo of a Greek helmet from Great Greece. 
It should recall Ulysses, the legendary, clever and subtle Greek hero whose deeds Homer sang in his poem, The Odyssey.
