SmartTracker4 Configuration

SmartTracker4 has several parameters you can change to control the output, efficiency and tracking speed. Here is the configuration dialog, with very typical (not necessarily optimal) settings.

  • Alignment Tolerance: the transverse alignment tolerance in micron (see Overview).
  • Bottom Layer: the lower layer of the trigger that is about to be inserted by using the Add button (see Overview).
  • Cells Along X: SmartTracker divides the volume in cells in order to speed up the search; in order not to decrease the efficiency, the cells should not be smaller than 20 µm, so their number along X must be set according to the size of the field of view.
  • Cells Along Y: SmartTracker divides the volume in cells in order to speed up the search; in order not to decrease the efficiency, the cells should not be smaller than 20 µm, so their number along Y must be set according to the size of the field of view.
  • Cell Overflow: the maximum number of grains a cell can contain in each layer.
  • Max Grains: the maximum number of grains computed over all layers.
  • Max Layers: the maximum number of layers that can be feeded to the tracker; should be at least equal to the number of layers sampled.
  • Max Pixels: grains formed by more pixels than MinArea are not considered for tracking.
  • Max Slope: the maximum slope being searched for. The tracking time increases quadratically with this parameter.
  • Min Pixels: grains formed by fewer pixels than MinArea are not considered for tracking.
  • Min Slope: the minimum slope being searched for. In order to avoid that dirt on the camera sensor generates perfectly vertical fake tracks, one can set this parameter to a small number, such as 0.001.
  • Min Grains at Slope = 0.1: the minimum number of grains for a track with slope = 0.1. This parameter helps to define the curve nmin(S) (see Overview).
  • Min Grains for Horizontal Tracks: the asymptotic minimum number of grains for very wide angle tracks. This parameter helps to define the curve nmin(S) (see Overview).
  • Min Grains for Vertical Tracks: the minimum number of grains a perfectly vertical track must have. This parameter helps to define the curve nmin(S) (see Overview).
  • Processors to use: the number of threads to use for tracking; this number should match the number of processors in the machine. The tracking speed scales then proportionally with the number of processors used. Increasing this number above the number of processor does not yield any benefit, and increases the thread handling overhead.
  • Top Layer: the upper layer of the trigger that is about to be inserted by using the Add button (see Overview).
  • Trigger Layers: list of the trigger layers for the trigger that is about to be inserted by using the Add button. These are the layers on which the track position and slope guessed from grains on Top Layer and Bottom Layer will be tested (see Overview).

Triggers are inserted only after the Add button is pressed and they appear in the list. In order to delete a trigger, select it on the list and press the Del button.

The Test button opens the testing dialog that can be used to tune the tracking speed by working on a set of grains stored in a file.
